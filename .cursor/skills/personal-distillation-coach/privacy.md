# 隐私与匿名化规范

本技能默认采用“可传播版本”写作标准：不暴露可识别真实主体的信息。

## 1) 默认脱敏范围
- 公司名称：统一替换为 `某互联网公司`、`某金融科技公司`、`某企业` 等。
- 具体团队名、业务线名：统一替换为 `某核心业务线`、`某项目组`。
- 非必要的精确地点、个人隐私细节：按语境弱化。

## 2) 可保留信息（通常安全）
- 技术方向与能力标签（后端、AI 工程化、Agent、测试等）
- 经验级别与经历阶段（一段/两段实习、校招准备中）
- 可公开的兴趣与价值观表达

## 3) 输出检查清单
- [ ] 是否出现真实公司/团队名称？
- [ ] 是否有可反向定位个人身份的细节组合？
- [ ] 是否存在“信息过度具体但对表达无增益”的内容？

若任一项为“是”，先脱敏再输出。
